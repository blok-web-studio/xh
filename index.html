01

curl vs xh

Same request. One is readable. The other requires a manual.

curl — 16 flags
curl -s -H "Accept: application/json" \
-H "Authorization: Bearer $TOKEN" \
-X POST https://api.example.com/data \
-d '{"name":"xh","type":"tool"}' \
| jq .

// still debugging flags
xh — 0 flags
xh POST https://api.example.com/data \
name=xh type=tool \
Authorization:"Bearer $TOKEN"

✓ HTTP/2.0 201 — coloured. readable. done.
02

What you get

Built for developers who want responses they can actually read.

📄

JSON by default

Every response is automatically formatted and coloured. No piping to jq.

🔐

Native auth

Bearer tokens, basic auth, API keys — inline, zero ceremony.

💾

Sessions

Auth once, reuse across requests. Cookies and headers persist.

🎨

Colourised output

Headers in cyan, keys blue, strings green. Readable at a glance.

⬇️

Downloads

Progress bars, resume, auto-filename. No extra tools.

⚡

Blazing fast

Single Rust binary. No Node, no Python. Zero setup.
